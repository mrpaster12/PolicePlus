## 1. معرفی کلی

PolicePlus یک پلاگین جامع و حرفه‌ای سیستم پلیس برای سرورهای ماینکرفت است. این پلاگین شامل سیستم‌های وانتد، زندان، دستبند، جایزه، قطب‌نما، حقوق، رتبه و بسیاری امکانات دیگر می‌باشد.

**ویژگی‌های کلیدی:**
- ✅ سیستم وانتد چندسطحی با کاهش خودکار
- ✅ سیستم زندان زمانی و معدنی (کار اجباری)
- ✅ سیستم دستبند با کشش فیزیکی (Elastic Drag)
- ✅ سیستم جایزه روی سر بازیکنان
- ✅ ردیابی با قطب‌نما و BossBar
- ✅ سیستم نمایش ستاره‌ای و عددی (تب‌لیست، زیر اسم، BossBar)
- ✅ پاداش اقتصادی با Vault
- ✅ سیستم حقوق برای پلیس‌ها
- ✅ سیستم رتبه‌بندی پلیس
- ✅ رابط گرافیکی (GUI) چندمرحله‌ای
- ✅ پشتیبانی از زبان‌های انگلیسی و فارسی
- ✅ سازگار با LuckPerms و PlaceholderAPI

## 2. سازگاری و نصب

**نسخه‌های سازگار:** Minecraft 1.13 تا 1.21.x
**سرورهای سازگار:** Spigot, Paper, Bukkit

**وابستگی‌ها:**
- `Vault` (اختیاری - برای سیستم اقتصاد)
- `PlaceholderAPI` (اختیاری - برای پلیس‌هولدرها)
- `CuffEm` (اختیاری - پلاگین دستبند خارجی)

**نصب:**
1. فایل `police-plus-2.5.0.jar` را در پوشه `plugins` قرار دهید
2. سرور را ری‌استارت کنید
3. فایل `config.yml` و فایل‌های زبان خودکار ساخته می‌شوند
4. تنظیمات را مطابق نیاز خود تغییر دهید
5. با دستور `/police reload` تنظیمات را ریلود کنید

## 4. سیستم پرمیشن (مجوزها)

### مجوزهای اصلی:
| پرمیشن | توضیح | پیش‌فرض |

| `policeplus.compass` | استفاده از قطب‌نما | OP |
| `policeplus.bypass` | دور زدن محدودیت‌های زندان | OP |

### مجوزهای جزئی (Granular):
| پرمیشن | توضیح |
|--------|--------|
| `policeplus.gui` | باز کردن رابط گرافیکی پلیس |
| `policeplus.handcuff.cuff` | دستبند زدن |
| `policeplus.handcuff.uncuff` | باز کردن دستبند |
| `policeplus.handcuff.give` | دادن آیتم دستبند |

| `policeplus.bounty.set` | تنظیم جایزه |
| `policeplus.bounty.remove` | حذف جایزه |
| `policeplus.rank.manage` | مدیریت رتبه‌ها |
| `policeplus.rank.set` | تنظیم رتبه |
| `policeplus.rank.view` | مشاهده رتبه‌ها |
| `policeplus.log.view` | مشاهده لاگ‌ها |
| `policeplus.log.clean` | پاکسازی لاگ‌ها |
| `policeplus.salary.view` | مشاهده حقوق |
| `policeplus.salary.pay` | پرداخت دستی حقوق |
| `policeplus.salary.admin` | مدیریت سیستم حقوق |
| `policeplus.stats.view` | مشاهده آمار |
| `policeplus.jail.execute` | زندانی کردن مظنون |
| `policeplus.jail.unjail` | آزاد کردن زندانی |

### سلسله مراتب پرمیشن:
```
policeplus.admin
├── policeplus.police (شامل تمام مجوزهای پلیسی)
│   ├── policeplus.gui
│   ├── policeplus.compass
│   ├── policeplus.handcuff.cuff
│   ├── policeplus.handcuff.uncuff
│   ├── policeplus.jail.execute
│   └── policeplus.jail.unjail
├── policeplus.wanted
└── policeplus.bypass
```

**نکته مهم:** سیستم پرمیشن PolicePlus از یک لایه دوگانه (Dual-Layer) استفاده می‌کند. اگر بازیکنی مجوز `policeplus.police` یا `policeplus.admin` یا OP داشته باشد، تمام مجوزهای جزئی به صورت خودکار اعطا می‌شوند. این سیستم کاملاً با LuckPerms سازگار است.

## 5. سیستم وانتد (Wanted)

### نحوه کار:
- بازیکنان می‌توانند توسط پلیس وانتد شوند
- هر بار کشتن بازیکن دیگر، سطح وانتد افزایش می‌یابد (قابل تنظیم)
- سطح وانتد به صورت خودکار کاهش می‌یابد (قابل تنظیم)
- سطح وانتد می‌تواند هنگام مرگ ریست شود (قابل تنظیم)

### تنظیمات:
```yaml
wanted:
  max_wanted_level: 5          # حداکثر سطح وانتد
  wanted_on_kill: true         # افزودن وانتد با کشتن
  wanted_level_per_kill: 1     # میزان افزوده‌شده برای هر کشتن
  auto_remove: true            # کاهش خودکار
  remove_interval_minutes: 30  # فاصله زمانی کاهش (دقیقه)
  reset_on_death:
    enabled: false             # ریست هنگام مرگ
    max_level_to_reset: 3      # فقط اگر وانتد کمتر یا مساوی این مقدار باشد
```

### هشدار زیبا به پلیس:
هنگامی که بازیکنی وانتد می‌شود، یک هشدار چندخطی زیبا با حاشیه و آیکون 🚔 به تمام پلیس‌های آنلاین ارسال می‌شود.

## 6. سیستم زندان (Jail)

### انواع زندان:
1. **زندان زمانی (TIME):** بازیکن باید مدت مشخصی در زندان بماند
2. **زندان معدنی (BLOCKS):** بازیکن باید تعداد مشخصی بلاک استخراج کند

### نحوه زندانی کردن:
1. پلیس بازیکن مظنون را دستگیر می‌کند (دستبند)
2. پلیس او را به نزدیکی زندان می‌برد
3. از رابط گرافیکی یا دستور `/police jail` استفاده می‌کند
4. مدت زمان و نوع زندان انتخاب می‌شود
5. بازیکن زندانی می‌شود

### قابلیت‌های زندان:
- ✅ محدودیت حرکت (ضد فرار)
- ✅ محدودیت شکستن/قرار دادن بلاک
- ✅ محدودیت استفاده از آیتم‌ها
- ✅ محدودیت تغییر دنیا (پورتال ندر)
- ✅ نوار اکشن نمایش زمان باقیمانده
- ✅ نوار اکشن نمایش بلاک‌های باقیمانده
- ✅ ابزار معدن (کلنگ زندان)

### تنظیمات:
```yaml
jail:
  type: "TIME"                # نوع زندان: TIME یا BLOCKS
  jail_time_per_wanted: 5     # دقیقه برای هر سطح وانتد
  max_jail_time: 60           # حداکثر زمان زندان (دقیقه)
  blocks_per_wanted: 100      # بلاک برای هر سطح وانتد
  allowed_blocks:             # بلاک‌های مجاز برای استخراج
    - "COBBLESTONE"
    - "STONE"
  arrest_distance: 10         # حداکثر فاصله دستگیری (بلاک)
  jail_required_distance: 7   # حداکثر فاصله از زندان
  clear_wanted_on_release: true  # پاک کردن وانتد هنگام آزادی
  default_jail_radius: 10     # شعاع محدودیت زندان
```

## 7. سیستم دستبند (Handcuff)

### نحوه کار:
- پلیس آیتم دستبند (Blaze Rod) را در دست می‌گیرد
- روی بازیکن مظنون راست‌کلیک می‌کند
- بازیکن دستبند می‌شود و حرکتش کند می‌شود
- سیستم کشش فیزیکی (Elastic Drag) مانع فرار می‌شود

### قابلیت‌ها:
- ✅ کشش فیزیکی (Elastic Drag) — بازیکن دستبند نمی‌تواند دور شود
- ✅ کاهش سرعت حرکت
- ✅ جلوگیری از شکستن/قرار دادن بلاک
- ✅ جلوگیری از استفاده از آیتم‌ها
- ✅ جلوگیری از پرتاب Ender Pearl
- ✅ جلوگیری از سوار شدن به وسایل نقلیه
- ✅ جلوگیری از تغییر دنیا
- ✅ ضرر ناپذیری کامل (حین کشش)
- ✅ نوار BossBar برای پلیس
- ✅ جلوگیری از برخورد فیزیکی (No Collision)
- ✅ قابلیت Combat-Logger (جلوگیری از فرار با قطع اتصال)
- ✅ محدودیت دستورات (فقط چت)
- ✅ زمان‌بندی خودکار باز شدن دستبند

### تنظیمات:
```yaml
handcuff:
  name: "cuff"                # نام نمایشی آیتم
  lore: "cuff"                # متن لور آیتم
  name_color: "&a"            # رنگ نام
  name_bold: true             # بولد بودن نام
  lore_color: "&7"            # رنگ لور
  max_time: 300               # حداکثر زمان دستبند (ثانیه)
  accept_plain_item: false    # پذیرفتن Blaze Rod معمولی
  apply_blindness: false      # اعمال اثر کوری
```

## 8. سیستم جایزه (Bounty)

### نحوه کار:
- هر بازیکنی می‌تواند روی سر بازیکن دیگر جایزه بگذارد
- جایزه از حساب بازیکن کسر می‌شود
- هر بازیکنی (پلیس یا شهروند) می‌تواند جایزه را با کشتن دریافت کند
- جایزه می‌تواند روی بازیکنان با وانتد 0 هم گذاشته شود

### قابلیت‌ها:
- ✅ گذاشتن جایزه روی هر بازیکنی (حتی بی‌گناه)
- ✅ افزودن به جایزه موجود
- ✅ حذف جایزه توسط ادمین
- ✅ دریافت جایزه با کشتن (پیام خصوصی به قاتل و قربانی)
- ✅ حذف خودکار جایزه هنگام زندانی شدن
- ✅ نمایش جایزه در رابط گرافیکی
- ✅ حذف بی‌صدا هنگام دریافت با کشتن (بدون پیام عمومی)

### پیام‌های دریافت جایزه:
- **به قاتل:** `[🚔 Bounty] شما با موفقیت {بازیکن} را کشتید و جایزه ${مبلغ} را دریافت کردید!`
- **به قربانی:** `[🚔 Bounty] شما توسط {بازیکن} کشته شدید و جایزه ${مبلغ} روی سر شما برداشته شد!`

### تنظیمات:
```yaml
bounty:
  enabled: true               # فعال‌سازی
  max_bounty: 10000           # حداکثر جایزه
  min_bounty: 10              # حداقل جایزه
  remove_on_jail: true        # حذف هنگام زندانی شدن
  currency_symbol: "$"        # نماد پول
```

## 10. سیستم نمایش (Display)

### حالات نمایش:
1. **ستاره‌ای (stars):** نمایش با ستاره‌های پر و خالی
2. **عددی (number):** نمایش عدد سطح وانتد

### محل‌های نمایش:
- ✅ تب‌لیست (Tab List)
- ✅ زیر اسم (Below Name)
- ✅ نوار BossBar

### تنظیمات:
```yaml
display:
  enabled: true
  mode: "stars"               # stars یا number
  stars:
    symbol: "★"               # نماد نمایش
    filled_color: "&6"        # رنگ ستاره پر
    empty_color: "&7"         # رنگ ستاره خالی
  tablist:
    enabled: true
    format_stars: " &7[&e{stars}&7]"
    format_number: " &7[&e{level}&7]"
  below_name:
    enabled: true
    objective: "wantedLevel"
    title: "&eWanted"
```

## 11. سیستم اقتصاد (Economy)

### نحوه کار:
- پلیس‌ها هنگام زندانی کردن مظنون پاداش نقدی دریافت می‌کنند
- مقدار پاداش = سطح وانتد × مبلغ به ازای هر سطح
- جایزه‌ها از حساب بازیکن کسر و به حساب قاتل واریز می‌شود
- از Vault استفاده می‌کند (سازگار با EssentialsX و سایر اقتصادها)

### تنظیمات:
```yaml
economy:
  enabled: true
  reward_per_wanted_level: 150.0  # پاداش به ازای هر سطح وانتد
```

## 12. سیستم حقوق (Salary)

### نحوه کار:
- پلیس‌ها بر اساس رتبه خود حقوق دریافت می‌کنند
- حقوق به صورت خودکار پرداخت می‌شود
- بازیکنان وانتد جریمه می‌شوند

### تنظیمات:
```yaml
salary:
  enabled: true

    enabled: true             # جریمه وانتد

```

## 16. رابط گرافیکی (GUI)

### رابط اصلی پلیس (`/police`):
- نمایش سرهای بازیکنان وانتد و دارای جایزه
- نمایش سطح وانتد و مبلغ جایزه در لور
- کلیک برای دستگیری یا ردیابی با قطب‌نما
- نمایش آمار کلی در آیتم اطلاعات

### رابط زندان (چندمرحله‌ای):
1. **مرحله اول:** انتخاب مدت زمان (5, 10, 15, 20 دقیقه یا سفارشی)
2. **مرحله دوم:** انتخاب نوع زندان (زمانی یا معدنی)
3. **مرحله سوم:** انتخاب محل زندان

## 18. سیستم زبان (Language)

### زبان‌های پشتیبانی‌شده:
- **انگلیسی (en):** `src/main/resources/languages/en.yml`
- **فارسی (fa):** `src/main/resources/languages/fa.yml`

### نحوه تغییر زبان:
1. فایل `config.yml` را باز کنید
2. مقدار `language` را به `"fa"` تغییر دهید
3. دستور `/police reload` را اجرا کنید

### سفارشی‌سازی پیام‌ها:
تمام پیام‌ها در فایل‌های زبان قابل سفارشی‌سازی هستند. از کدهای رنگ `&` استفاده کنید.
